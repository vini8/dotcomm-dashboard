6R Framework

R1

Retain
  • Keep host/application in source environment
  • Minimal analysis/validation of scope & application affinity

R4

Replatform
  • Up-version the OS/DB
  • Make use of Amazon Relational Database Service (Amazon RDS)
  • Some application changes
  • App reinstallation and cleanup

R2

Retire
  • Decommission
  • No migration of application

R5

Refactor
  • OS/DB porting
  • Middleware and app changes to use AWS services natively
  • Data conversion, DB transition (e.g., MySQL & Amazon Aurora)

R3

Rehost
  • Like-for-like app migration to AWS
  • Lift-and-shift approach
  • Minimal effort to make tha application work 'in' the cloud or take advantage of cloud native services

R6

Rearchitect
  • Application architecture changes
  • Using Software as a Service (SaaS) based offerings
  • Porting of application architecture

Best practices, methodology and guidelines for developers. These best practices are designed to enable applications to be built with portability and resilience when deployed to the web.


I. Codebase

| One codebase tracked in revision control, many deploys

II. Dependencies

| Explicitly declare and isolate dependencies

III. Config

| Store config in the environment

IV. Backing Services

| Treat backing services as attached resources

V. Build, Release, Run

| Strictly separate build and run stages

VI. Processes

| Execute the app as one or more stateless processes

VII. Port Binding

| Export services via port binding

VIII. Concurrency

| Scale-out via the process mode

IX. Disposability

| Maximize robustness with fast startup and graceful shutdown

X. Dev/Prod Parity

| Keep development, staging, and production as similar as possible

XI. Logs

| Treat logs as event streams

XII. Admin Processes

| Run admin/management tasks as one-off processes

Transform

These principles drive our technology future and lay out the vision for consolidated, cloud based, interoperable services that can transform County and City government service delivery.

All technology implementations should utilize Strategic Shared Services Platforms whenever possible.

If a Strategic Shared Services platform is not appropriate, a Managed Cloud solution that is interoperable with our Shared Services platforms should be chosen.

Give preference to solutions that work on any device, enable mobility and easily coordinate with other products.

Run

These principles are the bedrock of quality service delivery. These practices ensure a stable, secure and supportable technology environment that runs consistently.

We will achieve Service Level Agreements and Customer Satisfaction standards.

We will take all appropriate measures to ensure Stable, Secure and Supportable operating conditions as efficiently as possible.
